# E-Commerce Migration Guide

Guide for migrating the Acme Shop application from a traditional
layered architecture to Hexagonal Architecture with HexaGlue.

## Phase 1: Discovery

Run HexaGlue's audit plugin on the legacy codebase to identify anti-patterns:
anemic domain models, JPA annotations in business logic, missing port boundaries,
and primitive obsession across 50 classes.

## Phase 2: Hexagonal Restructuration

Reorganize the codebase into hexagonal packages:

- `domain/` — Pure domain model organized by bounded context (order, customer, product, payment, shipping, inventory)
- `ports/in/` — Driving ports (use case interfaces)
- `ports/out/` — Driven ports (repository and gateway interfaces)
- `application/` — Application services implementing use cases
- `infrastructure/` — External adapters (generated by HexaGlue)

## Phase 3: Domain Purification

Transform the anemic domain model into a rich one:

- Extract Value Objects: `Money`, `Address`, `Email`, `Quantity`
- Introduce Typed IDs: `OrderId`, `CustomerId`, `ProductId`, etc.
- Add factory methods and business logic to domain entities
- Remove all JPA annotations from domain classes

## Phase 4: Code Generation

Configure HexaGlue plugins to generate infrastructure:

- **JPA Plugin**: entities, repositories, mappers, adapters
- **REST Plugin**: controllers, DTOs, exception handler, configuration
- **Audit Plugin**: architecture compliance report
- **Living Doc Plugin**: architecture documentation

Configuration: see `hexagonal/hexaglue.yaml`.

## Key Architectural Rules

1. Domain classes have zero framework dependencies
2. Ports define the boundary between domain and infrastructure
3. Infrastructure adapters are generated, not hand-written
4. Application services orchestrate domain operations through ports
